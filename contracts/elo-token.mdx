`ELOToken` is a standard ERC20 token with a fixed, one-time mint. The entire supply is minted to the deployer at construction and immediately distributed to the predefined allocation addresses. No further minting is possible.

## Contract details

| Property        | Value                                          |

| Contract name   | `ELOToken`                                     |
| Solidity        | `^0.8.9`                                       |
| Inherits        | `Context`, `IERC20`, `IERC20Metadata`, `Ownable` |
| Token name      | `Effortless Order`                             |
| Symbol          | `ELO`                                          |
| Decimals        | `18`                                           |
| Total supply    | `1,000,000,000` (1B) ELO                       |
| BSC Testnet     | `0x6154818f854f5009C5CFa47213C68598E9dE02FA`  |

## Token distribution

The full supply is minted once in the constructor and immediately transferred to the allocation addresses. After the constructor runs, `mintingFinishedPermanent` is set to `true` and no further minting is allowed.

| Allocation          | Percentage | Amount (ELO)  |
| ------------------- | ---------- | ------------- |
| Presale             | 10%        | 100,000,000   |
| DEX                 | 10%        | 100,000,000   |
| CEX Reserved        | 15%        | 150,000,000   |
| Staking             | 25%        | 250,000,000   |
| Team                | 10%        | 100,000,000   |
| Order rewards       | 20%        | 200,000,000   |
| Customer rewards    | 5%         | 50,000,000    |
| Airdrop             | 5%         | 50,000,000    |
| **Total**           | **100%**   | **1,000,000,000** |

<Note>
The constructor asserts that all allocation amounts sum to the total supply. Deployment reverts if this check fails.
</Note>

## ERC20 functions

Standard functions from the `IERC20` interface:

```solidity
function totalSupply() external view returns (uint256);

function balanceOf(address account) external view returns (uint256);

function transfer(address recipient, uint256 amount) external returns (bool);

function allowance(address owner, address spender) external view returns (uint256);

function approve(address spender, uint256 amount) external returns (bool);

function transferFrom(
    address sender,
    address recipient,
    uint256 amount
) external returns (bool);
```

### Allowance helpers

```solidity
function increaseAllowance(address spender, uint256 addedValue) public virtual returns (bool);

function decreaseAllowance(address spender, uint256 subtractedValue) public virtual returns (bool);
```

These are safer alternatives to calling `approve` directly. Use `increaseAllowance` and `decreaseAllowance` to avoid the ERC20 double-spend race condition.

## Ownable functions

`ELOToken` extends the `Ownable` contract, which includes a time-lock mechanism in addition to standard ownership transfer.

```solidity
function owner() public view returns (address);

function transferOwnership(address newOwner) public virtual onlyOwner;

function renounceOwnership() public virtual onlyOwner;

// Locks the contract for owner for the given duration (in seconds)
function lock(uint256 time) public virtual onlyOwner;

// Unlocks the contract once the lock period has expired
function unlock() public virtual;

function geUnlockTime() public view returns (uint256);
```

`lock(uint256 time)` sets the owner to `address(0)` and records `block.timestamp + time` as the unlock time. Only the previous owner can call `unlock()` after the lock period has elapsed.

## Events

```solidity
event Transfer(address indexed from, address indexed to, uint256 value);

event Approval(address indexed owner, address indexed spender, uint256 value);

event OwnershipTransferred(address indexed previousOwner, address indexed newOwner);
```

`Transfer` is emitted on every token transfer, including during the initial mint (where `from` is `address(0)`) and any burn (where `to` is `address(0)`).
