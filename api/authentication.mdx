title: "Authentication"
description: "Register, log in, and manage users through the ELO Token Demo auth API"
---

All authentication endpoints are served under the base path `/presale/auth/`.

<Note>
  JWT tokens expire after **1 hour**. Your client must re-authenticate to obtain a fresh token when the current one expires.
</Note>

## Using the token

Include the JWT in the `Authorization` header for all protected requests:

```
Authorization: Bearer <token>
```

---

## POST /presale/auth/register

Create a new user account. Returns a signed JWT on success.

### Request body

<ParamField body="username" type="string" required>
  Unique username for the account.
</ParamField>

<ParamField body="email" type="string" required>
  Unique email address for the account.
</ParamField>

<ParamField body="password" type="string" required>
  Plain-text password. Stored as a bcrypt hash (10 rounds).
</ParamField>

### Response

<ResponseField name="status" type="string" required>
  `"success"` on successful registration.
</ResponseField>

<ResponseField name="user" type="object" required>
  <Expandable title="properties">
    <ResponseField name="username" type="string">
      The registered username.
    </ResponseField>
    <ResponseField name="email" type="string">
      The registered email address.
    </ResponseField>
    <ResponseField name="token" type="string">
      Signed JWT. Valid for 1 hour.
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of account creation.
    </ResponseField>
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of last update.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:8082/presale/auth/register \
  --header 'Content-Type: application/json' \
  --data '{
    "username": "alice",
    "email": "alice@example.com",
    "password": "s3cur3p@ss"
  }'
```

```json Response
{
  "status": "success",
  "user": {
    "_id": "64a1f2c3e4b0a1b2c3d4e5f6",
    "username": "alice",
    "email": "alice@example.com",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "createdAt": "2024-01-15T10:30:00.000Z",
    "updatedAt": "2024-01-15T10:30:00.000Z"
  }
}
```

</CodeGroup>

### Error responses

| Code | Message |
| ---- | ------- |
| `202` | `"All input is required."` — one or more fields are missing |
| `201` | `"Username Already Exist."` — username is already taken |
| `201` | `"Email Already Exist."` — email is already registered |
| `400` | Unexpected server error |

## POST /presale/auth/login

Authenticate an existing user. Returns a signed JWT on success.

### Request body

<ParamField body="email" type="string" required>
  The user's email address **or** username. Both are accepted.
</ParamField>

<ParamField body="password" type="string" required>
  The user's plain-text password.
</ParamField>

### Response

<ResponseField name="status" type="string" required>
  `"success"` on successful authentication.
</ResponseField>

<ResponseField name="user" type="object" required>
  <Expandable title="properties">
    <ResponseField name="username" type="string">
      The user's username.
    </ResponseField>
    <ResponseField name="email" type="string">
      The user's email address.
    </ResponseField>
    <ResponseField name="token" type="string">
      Signed JWT. Valid for 1 hour.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>

```bash cURL (email)
curl --request POST \
  --url http://localhost:8082/presale/auth/login \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "alice@example.com",
    "password": "s3cur3p@ss"
  }'
```

```bash cURL (username)
curl --request POST \
  --url http://localhost:8082/presale/auth/login \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "alice",
    "password": "s3cur3p@ss"
  }'
```

```json Response
{
  "status": "success",
  "user": {
    "_id": "64a1f2c3e4b0a1b2c3d4e5f6",
    "username": "alice",
    "email": "alice@example.com",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
```

</CodeGroup>

### Error responses

| Code | Message |
| ---- | ------- |
| `202` | `"All input is required."` — email or password is missing |
| `203` | `"Invalid Credentials."` — no matching user or wrong password |
| `400` | Unexpected server error |

## POST /presale/auth/update

Update the password for an existing account.

### Request body

<ParamField body="email" type="string" required>
  The user's email address **or** username.
</ParamField>

<ParamField body="oldpass" type="string" required>
  The user's current password, used for verification.
</ParamField>

<ParamField body="newpass" type="string" required>
  The new password to set. Stored as a bcrypt hash (10 rounds).
</ParamField>

### Response

<ResponseField name="status" type="string" required>
  `"success"` when the password was updated.
</ResponseField>

<ResponseField name="user" type="object" required>
  The updated user object (password field reflects the previous hash at response time).
</ResponseField>

### Example

<CodeGroup>

```bash cURL
curl --request POST \
  --url http://localhost:8082/presale/auth/update \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "alice@example.com",
    "oldpass": "s3cur3p@ss",
    "newpass": "n3wp@ssw0rd"
  }'
```

```json Response
{
  "status": "success",
  "user": {
    "_id": "64a1f2c3e4b0a1b2c3d4e5f6",
    "username": "alice",
    "email": "alice@example.com"
  }
}
```

</CodeGroup>

### Error responses

| Code | Message |
| ---- | ------- |
| `202` | `"All input is required."` — one or more fields are missing |
| `203` | `"Invalid Credentials."` — no matching user or `oldpass` is incorrect |
| `400` | Unexpected server error |
