## Prerequisites

Before you begin, make sure you have the following installed and ready:

- **Node.js v16 or later** — required for both the Express backend and the React frontend
- **MetaMask or a compatible Web3 wallet** — used to connect to the app and sign transactions
- **MongoDB** — the backend uses MongoDB (via Mongoose) to store presale data; a [MongoDB Atlas](https://www.mongodb.com/atlas) free tier cluster works fine

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the ELO Token Demo repository to your local machine:

    ```bash
    git clone https://github.com/mega4010/TokenDemo.git
    cd TokenDemo
    ```
  </Step>

  <Step title="Install dependencies">
    Install all frontend and backend packages from the project root. Both the React app and Express server share the same `package.json`.

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Create a `config.env` file in the project root with your MongoDB connection string and JWT secret. See [Environment setup](/environment-setup) for full details.

    ```bash config.env
    ATLAS_URI=mongodb+srv://<user>:<password>@<cluster>.mongodb.net/presale?retryWrites=true&w=majority
    PORT=8082
    TOKEN_KEY=your-secret-key
    ```
  </Step>

  <Step title="Start the app">
    Run the following command to start both the backend and frontend together:

    ```bash
    npm start
    ```

    <Note>
      `npm start` runs both the Express server and the React dev server concurrently using a single command: `npm run server | react-scripts --openssl-legacy-provider start`. You do not need to start them separately.
    </Note>
  </Step>
</Steps>

## Running locally

Once started, two servers are available:

| Server | URL |
|---|---|
| React frontend | `http://localhost:3000` |
| Express backend | `http://localhost:8082` |

Open `http://localhost:3000` in your browser to access the application.

### Available npm scripts

| Command | Description |
|---|---|
| `npm start` | Start Express backend and React dev server together |
| `npm run server` | Run Express backend only |
| `npm run build` | Build React app for production |
| `npm run deploy` | Deploy contracts via Truffle |
| `npm run devchain` | Start a local Truffle development chain |

## Connecting your wallet

On your first visit to `http://localhost:3000`, the app prompts you to connect a wallet before accessing any DeFi features.

<Steps>
  <Step title="Open the app">
    Navigate to `http://localhost:3000`. You will land on the home page.
  </Step>

  <Step title="Click Connect Wallet">
    Click the **Connect Wallet** button in the navigation bar. A wallet selection dialog appears.
  </Step>

  <Step title="Select your wallet">
    Choose **MetaMask**, **Rabby**, or any other injected Web3 wallet. The app uses `@web3-react/injected-connector` to detect browser-injected wallets automatically.
  </Step>

  <Step title="Approve the connection">
    Confirm the connection request in your wallet extension. Once connected, your wallet address appears in the navigation bar.
  </Step>

  <Step title="Switch to BSC Testnet">
    ELO Token Demo is deployed on **BSC Testnet** (chain ID 97). Switch your wallet network to BSC Testnet to interact with the presale and token contracts. Your wallet may prompt you to add the network if it is not already configured.
  </Step>
</Steps>

## Quick tour of the app

The React app is a single-page application with the following routes:

| Route | Description |
|---|---|
| `/` | Landing page |
| `/home` | Main dashboard after connecting |
| `/pre-sale` | ELO token presale launchpad |
| `/swap` | Token swap interface |
| `/mint` | NFT minting |
| `/nfts` | NFT gallery |
| `/stake` | Staking UI |
| `/transactions` | Transaction history |
| `/onramp` | Fiat-to-crypto on-ramp (Ramper integration) |

<Note>
  Staking (`/stake`) is a UI placeholder — the staking smart contract is planned but not yet deployed.
</Note>
